# Дизайн: installer UX и визуальная доработка иконки яркости

## Цель

Улучшить пользовательский опыт для `omarchy-waybar-brightness` в двух местах:

- installer должен уметь подготовить типовую свежую Omarchy-систему для DDC/CI на внешнем мониторе
- иконка солнца в Waybar должна выглядеть заметно, а не как маленькая точка

## Изменение 1: self-healing installer

`install.sh` должен не только копировать файлы модуля и патчить Waybar-конфиг, но и выполнять минимальную системную подготовку для внешнего монитора.

Ожидаемое поведение:

1. проверить наличие `ddcutil`
2. если `ddcutil` отсутствует, попытаться установить его через `omarchy-pkg-add ddcutil`
3. если доступен `modprobe`, попытаться загрузить `i2c-dev`
4. выполнить короткую DDC-проверку:
   - `ddcutil detect --brief`
   - опционально `ddcutil getvcp 10`, если обнаружен монитор
5. если DDC готов, вывести короткое подтверждение
6. если DDC не готов, не прерывать установку модуля, а печатать понятную диагностику

Ограничения:

- installer не должен становиться хрупким или необратимым
- если `sudo` недоступен или пакет не ставится, установка самого Waybar-модуля все равно должна завершаться
- нельзя трогать `~/.local/share/omarchy/`
- системная подготовка должна быть идемпотентной

Почему так:

- это соответствует переносимой Omarchy-фиче
- это лучший UX на свежей системе
- это избавляет пользователя от сценария "модуль установлен, но молча серый"

## Изменение 2: заметная иконка солнца

`waybar/brightness/brightness.css` должен сделать модуль визуально читаемым.

Ожидаемое поведение:

- иконка солнца выглядит как иконка, а не как точка
- модуль сохраняет компактность и не ломает существующий ритм Waybar
- `unsupported` состояние остается более тусклым, чем `active`

Минимальный путь:

- увеличить `font-size`
- увеличить `min-width`
- при необходимости слегка скорректировать `font-weight` или spacing, но без агрессивного вида

## Тестирование

Нужно проверить:

- installer не ломается, если `ddcutil` уже установлен
- installer корректно проходит, если `ddcutil` отсутствует, но `omarchy-pkg-add` доступен
- installer не падает, если системная подготовка DDC не удалась
- Waybar-модуль после установки остается рабочим
- CSS по скриншоту/живому виду делает солнце заметным

## Что не меняем

- архитектуру модуля
- способ выбора устройства яркости
- scroll-модель управления
- формат JSON для Waybar
